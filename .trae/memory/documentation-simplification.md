# 文档简化任务记忆

## 任务概述

简化 README.md 和 AGENTS.md 文档，使其更精炼、更有效。

## 当前问题分析

### README.md (1377行)
- 内容过于冗长，包含大量重复信息
- 代码示例过多，应该放在独立文档
- 配置示例过于详细，应该链接到配置文档
- API文档应该独立，不应该在README中详细展开

### AGENTS.md (1333行)
- 很多内容与README重复
- 模块说明过于详细
- 构建和发布流程应该独立文档
- 代码规范部分过于冗长

## 简化策略

### README.md 简化后应保留
1. 项目简介和徽章
2. 核心特性概览（精简列表）
3. 快速开始（安装和运行）
4. 简化的项目结构
5. 文档索引链接
6. 许可证

### AGENTS.md 简化后应保留
1. 项目概述
2. 简化的项目结构
3. 核心模块简要说明（列表形式）
4. 配置文件位置表格
5. 开发规范（精简版）
6. 版本历史

## 执行路径

1. 创建任务记忆文档
2. 重写 README.md - 精简为核心内容
3. 重写 AGENTS.md - 精简为开发者指南
4. 更新 CHANGELOG.md
5. 审查和验证

## 预期效果

- README.md: 从1377行精简到约200-300行
- AGENTS.md: 从1333行精简到约300-400行
- 保留所有关键信息，移除冗余内容
- 通过链接指向详细文档

## 反思

- 文档应该遵循"单一职责"原则，README负责介绍和快速开始，AGENTS负责开发指南
- 详细内容应该分散到独立的文档中
- 避免在多个文档中重复相同的内容
- 使用表格形式展示信息更加紧凑
- 通过链接指向详细文档，保持主文档精炼
