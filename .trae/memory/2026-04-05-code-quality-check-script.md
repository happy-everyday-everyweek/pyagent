# 代码质量检查脚本实现

## 任务执行路径

1. **了解项目当前测试结构和配置**
   - 查看 `tests/` 目录结构，发现已有 18 个测试文件
   - 查看 `pyproject.toml` 中的测试配置
   - 查看 `pytest.ini` 配置文件
   - 查看 `conftest.py` 测试配置

2. **了解项目当前代码风格检查配置**
   - 发现项目已配置 ruff 和 mypy
   - 但配置较为简单，只启用了少量规则
   - 没有独立的检查脚本

3. **编写运行测试和代码风格检查的脚本**
   - 创建 `check.ps1` PowerShell 脚本
   - 支持运行 pytest 测试
   - 支持运行 ruff 代码风格检查
   - 支持运行 mypy 类型检查
   - 支持运行 bandit 安全检查
   - 支持运行 pydocstyle 文档字符串检查
   - 支持自动安装开发依赖
   - 支持多种参数组合

4. **引入更多代码风格检查工具**
   - 在 `pyproject.toml` 中添加 bandit 和 pydocstyle 依赖
   - 扩展 ruff 规则集，从 8 个规则集扩展到 25 个
   - 添加 bandit 配置（安全检查）
   - 添加 pydocstyle 配置（文档字符串检查）
   - 添加 ruff 格式化配置

5. **更新智能体文档**
   - 在 `AGENTS.md` 中添加"代码质量检查"章节
   - 详细说明检查工具的使用方法
   - 说明 Ruff 规则集
   - 提供添加新测试的示例
