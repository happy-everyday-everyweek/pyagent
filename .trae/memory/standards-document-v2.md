# 任务记忆：细化文档规范和版本号更新说明

## 任务概述

扩展项目规范文档，细化各类文档的格式规范，添加不同类型文档必须包含的内容说明，并新增版本号更新时机和详细说明章节。

---

## 任务执行路径

1. **扩展文档规范**
   - 添加文档类型总览表格
   - 新增 README 文档规范（必须包含的内容、完整模板）
   - 新增 CHANGELOG 文档规范（变更类型说明、完整模板）
   - 新增架构文档规范（必须包含的内容、架构图示例）
   - 扩展 API 文档规范（详细的参数说明、状态码、调用示例）
   - 扩展模块文档规范（完整的 API 参考、配置说明、示例代码）
   - 新增配置文档规范（环境变量、配置项说明、默认值）
   - 新增部署文档规范（环境要求、安装步骤、Systemd 服务）
   - 新增故障排查文档规范（常见问题、错误代码、排查步骤）
   - 保留 Markdown 通用格式规范

2. **添加版本号更新说明**
   - 版本号更新时机表格
   - MAJOR 版本更新详细说明（四种情况）
   - MINOR 版本更新详细说明（三种情况）
   - PATCH 版本更新详细说明（四种情况）
   - 版本号更新流程（四步流程）
   - 预发布版本规范（Alpha/Beta/RC）
   - 版本兼容性说明格式

3. **更新 AGENTS.md**
   - 扩展文档规范概要部分
   - 添加文档类型表格
   - 添加各类文档必须包含的内容说明
   - 扩展版本号规范概要
   - 添加版本号更新时机表格
   - 添加各版本类型更新条件
   - 添加预发布版本说明

4. **更新 CHANGELOG.md**
   - 添加 v0.9.1 版本记录（文档规范细化）
   - 添加 v0.9.2 版本记录（版本号更新规范）

---

## 任务执行路径可优化的地方

1. **文档模板**
   - 可以将文档模板提取为独立文件，便于复制使用
   - 可以添加文档生成脚本

2. **自动化检查**
   - 可以添加文档格式检查工具
   - 可以添加 CHANGELOG 格式验证

3. **版本管理**
   - 可以添加版本号自动更新脚本
   - 可以添加 Git 标签自动创建脚本
