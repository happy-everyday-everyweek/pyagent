# PyAgent v0.8.0 - 人类任务管理系统实现

**日期**: 2026-04-02
**版本**: v0.8.0
**任务**: Task 6 - 人类任务管理系统

## 任务概述

实现为人类用户设计的任务管理系统（区别于AI Todo系统），包括任务创建、分类管理、提醒通知等完整功能。

## 实现路径

### 1. 核心模块创建

#### 1.1 任务数据模型 (src/human_tasks/task.py)
- 创建 `TaskStatus` 枚举：pending, in_progress, completed, cancelled
- 创建 `Priority` 枚举：low, medium, high, urgent
- 创建 `SubTask` 数据类：支持子任务管理
- 创建 `HumanTask` 数据类：
  - 基本信息：task_id, title, description, category, tags
  - 状态管理：status, priority
  - 时间管理：due_date, reminder, created_at, updated_at, completed_at
  - 子任务和附件：subtasks, attachments
  - 辅助方法：is_overdue(), is_due_today(), get_progress(), complete_subtask()等

#### 1.2 分类管理 (src/human_tasks/category.py)
- 创建 `Category` 数据类：id, name, color, icon, description
- 创建 `CategoryManager` 类：
  - 默认分类初始化：工作、个人、学习、健康、其他
  - CRUD操作：create_category(), update_category(), delete_category()
  - 查询方法：get_category(), list_categories(), get_statistics()
  - 数据持久化：JSON文件存储

#### 1.3 任务管理器 (src/human_tasks/manager.py)
- 创建 `TaskManager` 类：
  - 任务CRUD：create_task(), update_task(), delete_task()
  - 状态管理：complete_task(), cancel_task(), start_task()
  - 查询功能：get_task(), list_tasks(), search_tasks()
  - 特殊查询：get_overdue_tasks(), get_tasks_due_today(), get_tasks_by_category()
  - 统计功能：get_statistics()
  - 子任务管理：add_subtask(), complete_subtask(), remove_subtask()
  - 数据持久化：JSON文件存储

#### 1.4 通知服务 (src/human_tasks/notification.py)
- 创建 `NotificationType` 枚举：reminder, due_soon, overdue, completed, cancelled
- 创建 `Notification` 数据类：id, task_id, type, title, message, scheduled_at, sent_at, sent
- 创建 `NotificationService` 类：
  - 提醒管理：schedule_reminder(), cancel_reminder()
  - 通知发送：send_notification()
  - 后台检查：check_due_reminders(), start_background_check(), stop_background_check()
  - 处理器注册：register_handler(), unregister_handler()
  - 数据持久化：JSON文件存储

#### 1.5 模块导出 (src/human_tasks/__init__.py)
- 导出所有公共类和实例
- 提供清晰的模块文档说明

### 2. 配置文件

#### 2.1 任务管理配置 (config/human_tasks.yaml)
- 数据存储配置：data_dir, auto_save, save_interval
- 任务默认配置：default_priority, default_category, 最大限制
- 分类配置：enable_default_categories, max_categories
- 提醒配置：enabled, check_interval, advance_reminders
- 通知配置：enabled, max_pending, retention_days
- UI配置：show_completed_tasks, default_view, sort_by
- 导出配置：formats, include_completed
- 统计配置：enable_auto_stats, stats_interval

### 3. API路由

#### 3.1 Web API (src/web/routes/human_tasks_routes.py)
- 任务管理API：
  - POST /api/human-tasks/tasks - 创建任务
  - GET /api/human-tasks/tasks - 列出任务（支持过滤）
  - GET /api/human-tasks/tasks/{task_id} - 获取任务详情
  - PUT /api/human-tasks/tasks/{task_id} - 更新任务
  - DELETE /api/human-tasks/tasks/{task_id} - 删除任务
  - POST /api/human-tasks/tasks/{task_id}/complete - 完成任务
  - POST /api/human-tasks/tasks/{task_id}/cancel - 取消任务
  - POST /api/human-tasks/tasks/{task_id}/start - 开始任务
  
- 子任务API：
  - POST /api/human-tasks/tasks/{task_id}/subtasks - 添加子任务
  - POST /api/human-tasks/tasks/{task_id}/subtasks/{subtask_id}/complete - 完成子任务
  - DELETE /api/human-tasks/tasks/{task_id}/subtasks/{subtask_id} - 移除子任务
  
- 查询API：
  - GET /api/human-tasks/tasks/search/{query} - 搜索任务
  - GET /api/human-tasks/tasks/overdue - 获取过期任务
  - GET /api/human-tasks/tasks/due-today - 获取今天到期任务
  - GET /api/human-tasks/statistics - 获取统计信息
  
- 分类API：
  - POST /api/human-tasks/categories - 创建分类
  - GET /api/human-tasks/categories - 列出所有分类
  - GET /api/human-tasks/categories/{category_id} - 获取分类详情
  - PUT /api/human-tasks/categories/{category_id} - 更新分类
  - DELETE /api/human-tasks/categories/{category_id} - 删除分类
  - GET /api/human-tasks/categories/{category_id}/tasks - 获取分类下的任务
  
- 提醒通知API：
  - POST /api/human-tasks/tasks/{task_id}/reminders - 安排提醒
  - DELETE /api/human-tasks/tasks/{task_id}/reminders - 取消提醒
  - POST /api/human-tasks/tasks/{task_id}/notifications - 发送通知
  - GET /api/human-tasks/notifications/pending - 获取待发送通知
  - POST /api/human-tasks/notifications/check - 检查到期提醒

#### 3.2 应用集成 (src/web/app.py)
- 导入 human_tasks_router
- 注册路由到 FastAPI 应用

## 技术特点

### 1. 数据模型设计
- 使用 dataclass 简化数据类定义
- 使用 Enum 定义状态和类型
- 提供完整的 to_dict() 和 from_dict() 方法支持序列化
- 支持进度计算和状态检查

### 2. 持久化存储
- 使用 JSON 文件存储数据
- 自动创建数据目录
- 支持自动保存和手动保存
- 数据加载时容错处理

### 3. 异步支持
- 通知服务支持异步后台检查
- 使用 asyncio.Task 管理后台任务
- 支持优雅启动和停止

### 4. API设计
- 遵循 RESTful 设计原则
- 使用 Pydantic 模型验证请求
- 统一的错误处理和响应格式
- 完整的 HTTP 状态码使用

## 与AI Todo系统的区别

| 特性 | AI Todo系统 | 人类任务系统 |
|------|------------|------------|
| 目标用户 | AI智能体 | 人类用户 |
| 任务层级 | 三级（阶段/任务/步骤） | 两级（任务/子任务） |
| 时间管理 | 无 | 截止日期、提醒 |
| 分类系统 | 无 | 多分类、自定义颜色图标 |
| 通知系统 | 无 | 完整的提醒和通知 |
| 验收文档 | 自动生成 | 无 |
| 反思机制 | 自动反思 | 无 |
