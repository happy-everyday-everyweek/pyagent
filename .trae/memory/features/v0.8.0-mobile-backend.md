### 3. 核心类实现

#### LinuxEnv 类
- 管理嵌入式Linux环境
- 检测环境信息（Python版本、Linux发行版、架构等）
- 启动/停止Python后端服务
- 提供本地API端点
- 健康检查功能

#### MobileBackend 类
- 单例模式实现
- 检测是否运行在移动设备上
- 检测设备能力（屏幕、相机、GPS等）
- 初始化和管理移动端后端
- 设置移动端特定API路由
- 支持三种后端模式：standalone、embedded、client

#### ScreenTools 类
- 继承 UnifiedTool 基类
- 实现三阶段生命周期（activate/execute/dormant）
- 截图功能（capture_screen）
- 点击操作（tap）
- 滑动操作（swipe）
- 文本输入（input_text）
- 按键操作（press_key）
- 长按操作（long_press）
- 通过ADB与Android设备交互

#### NotificationReader 类
- 读取设备通知
- 支持通知过滤（包名、类别、优先级）
- 清除通知功能
- 解析通知数据

#### NotificationListener 类
- 实时监听新通知
- 支持回调函数注册
- 可配置检查间隔

#### SMSTools 类
- 继承 UnifiedTool 基类
- 读取短信消息
- 发送短信
- 获取未读消息
- 标记已读
- 删除消息
- 搜索短信
- 获取对话线程

### 4. 数据结构定义

#### 枚举类型
- EnvironmentStatus: 环境状态
- BackendMode: 后端模式
- BackendStatus: 后端状态
- NotificationPriority: 通知优先级
- NotificationCategory: 通知类别
- MessageType: 消息类型
- MessageStatus: 消息状态

#### 数据类
- EnvironmentInfo: 环境信息
- ServiceConfig: 服务配置
- MobileCapabilities: 移动设备能力
- MobileConfig: 移动端配置
- ScreenInfo: 屏幕信息
- GestureResult: 手势操作结果
- Notification: 通知数据结构
- NotificationFilter: 通知过滤器
- SMSMessage: 短信消息
- SMSFilter: 短信过滤器

## 技术要点

### 1. 统一工具接口
- ScreenTools 和 SMSTools 继承自 UnifiedTool 基类
- 实现三阶段生命周期管理
- 支持状态转换验证
- 提供统一的调用入口

### 2. 移动设备检测
- 通过 platform 模块检测系统类型
- 检查 Android 特征文件路径
- 支持自动检测设备能力

### 3. ADB 交互
- 使用 subprocess 调用 ADB 命令
- 支持屏幕操作、通知读取、短信操作
- 处理命令输出解析

### 4. 异步编程
- 所有操作使用 async/await
- 支持异步健康检查
- 支持异步通知监听
