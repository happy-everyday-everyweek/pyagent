# 视频编辑器管理器系统重构

## 任务概述

参考 Cutia 项目，重构视频编辑器核心，集成命令系统，创建 SelectionManager 和 SaveManager。

## 完成路径

### 1. 分析现有代码结构
- 查看了 `src/video/editor_core.py` 现有实现
- 查看了 `src/video/commands/` 命令系统实现
- 参考了 Cutia 的 `selection-manager.ts` 和 `save-manager.ts`

### 2. 创建 managers 目录结构
```
src/video/managers/
├── __init__.py
├── selection.py
└── save.py
```

### 3. 实现 SelectionManager
- 使用 Set[str] 存储选中的元素和轨道 ID
- 支持单选和多选模式
- 提供订阅机制通知选择变化
- 通过 project 参数获取选中的对象

### 4. 实现 SaveManager
- 支持自动保存和手动保存
- 使用 debounce 机制避免频繁保存
- 支持暂停/恢复保存
- 线程安全的保存操作

### 5. 重构 EditorCore
- 集成 CommandManager、SelectionManager、SaveManager
- TimelineManager 使用 SelectionManager 管理选择状态
- 添加 undo()/redo() 快捷方法
- 执行命令时自动标记脏状态

## 反思与优化建议

### 已完成
1. SelectionManager 实现了完整的选择管理功能
2. SaveManager 实现了自动保存和手动保存
3. EditorCore 成功集成了命令系统

### 可优化方向
1. **命令实现**: 可以为常见操作（如移动元素、删除元素）创建具体的命令类
2. **事件系统**: 可以添加更完善的事件系统，支持更细粒度的状态变化通知
3. **持久化**: SaveManager 可以支持更多的存储后端（如数据库）
4. **选择框**: 可以添加框选功能，支持区域选择多个元素

### 兼容性考虑
- TimelineState 中的 selected_elements 和 selected_tracks 已移除
- TimelineManager 的选择方法已移除，改用 SelectionManager
- 现有代码如需使用选择功能，需通过 editor.selection 访问
