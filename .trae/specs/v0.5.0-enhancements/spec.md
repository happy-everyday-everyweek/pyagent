# v0.5.0 功能增强 Spec

## Why
v0.4.0架构重构完成后，需要增强任务状态管理、优化UI界面显示、添加热更新功能，提升用户体验和系统可维护性。

## What Changes
- 新增任务四状态系统：活跃、暂停、异常、等待
- 优化UI界面任务卡片显示
- 新增热更新功能

## Impact
- Affected specs: 任务系统、Web UI、执行模块
- Affected code:
  - `src/execution/task.py` - 任务状态扩展
  - `frontend/src/views/TasksView.vue` - UI优化
  - `src/web/routes/task_routes.py` - 热更新API
  - 新增 `src/web/routes/hot_reload.py` - 热更新模块

## ADDED Requirements

### Requirement: 任务四状态系统
系统 SHALL 支持任务的四种状态管理。

#### Scenario: 活跃状态（默认）
- **WHEN** 任务创建时
- **THEN** 任务状态默认为"活跃"，正常执行

#### Scenario: 暂停状态
- **WHEN** 系统资源不足或用户手动暂停
- **THEN** 任务状态变为"暂停"，暂停执行

#### Scenario: 异常状态
- **WHEN** AI在非常多次数尝试后依旧无法完成任务
- **THEN** 任务状态变为"异常"，等待人工介入

#### Scenario: 等待状态
- **WHEN** 任务需要用户操作
- **THEN** 任务状态变为"等待"，等待用户确认或协助

### Requirement: UI任务卡片显示
系统 SHALL 在UI界面以卡片形式展示任务状态。

#### Scenario: 执行中状态显示
- **WHEN** 任务正在执行
- **THEN** 显示"执行｜规划中"或"执行｜XX%"

#### Scenario: 等待用户状态显示
- **WHEN** 任务需要用户操作
- **THEN** 显示"执行｜须您确认"或"执行｜须您协助"

#### Scenario: 终态显示
- **WHEN** 任务完成或失败
- **THEN** 显示"执行｜完成"或"执行｜失败"

### Requirement: 热更新功能
系统 SHALL 支持通过导入zip压缩包进行热更新，无需重启服务。

#### Scenario: 上传更新包
- **WHEN** 用户上传新版本的zip压缩包
- **THEN** 系统自动解压、验证、应用更新

#### Scenario: 无缝更新
- **WHEN** 更新应用成功
- **THEN** 服务无需重启，新功能立即生效

## MODIFIED Requirements

### Requirement: 任务状态枚举
任务状态枚举 SHALL 扩展为：
```python
class TaskState(Enum):
    ACTIVE = "active"      # 活跃（默认）
    PAUSED = "paused"      # 暂停
    ERROR = "error"        # 异常
    WAITING = "waiting"    # 等待用户操作
```

### Requirement: 任务进度显示
任务卡片 SHALL 支持以下显示类型：
- 执行｜规划中
- 执行｜10%（进度百分比）
- 执行｜须您确认
- 执行｜须您协助
- 执行｜完成
- 执行｜失败

## REMOVED Requirements
无移除的需求。

## 状态设计

```
任务状态流转:

创建 → 活跃(ACTIVE)
       ↓
    执行中 ──────→ 暂停(PAUSED) ──→ 活跃
       │              ↑
       ↓              │
    等待(WAITING) ←───┘
       │
       ↓
    异常(ERROR) ──→ 人工介入 ──→ 活跃
       │
       ↓
    完成/失败
```

## 热更新流程

```
用户上传zip → 验证签名 → 解压到临时目录
     ↓
备份当前版本 → 复制新文件 → 更新模块
     ↓
清理临时文件 → 返回更新结果
```
